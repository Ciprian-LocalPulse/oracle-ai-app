# Security Policy

## Supported Scope

This public repository may include static application presentation files and governance documentation for Oracle AI App. Security review for the Public Edition is limited to files intentionally published in this repository.

The following issues are in scope:

- Public repository content that accidentally exposes secrets or sensitive configuration.
- Public documentation that discloses private implementation details.
- Static page behavior that could create browser security concerns.
- Public links that route users to unsafe or unofficial destinations.
- Governance language that creates avoidable security, legal, or procurement risk.

## Out of Scope

The following items are outside the scope of this public repository:

- Private backend services, API routes, databases, and model orchestration.
- Authentication, subscription, payment, credit, and token-generation systems.
- Client deployments, customer environments, or internal infrastructure.
- Proprietary prompts, playbooks, telemetry, analytics, monitoring, or response logic.
- Social engineering, denial-of-service testing, phishing, or intrusive scanning.

Do not attempt to access private accounts, systems, repositories, databases, or data without explicit written authorization.

## Reporting a Vulnerability

Please report sensitive concerns privately by email:

**contact@localpulse.pro**

Include:

- A concise description of the concern.
- The affected public file, link, or repository artifact.
- Non-intrusive reproduction details, if applicable.
- Potential impact.
- Suggested remediation, if known.

Do not include secrets, customer data, private keys, tokens, exploit payloads, or third-party confidential information in the report.

## Response Expectations

Reports are reviewed on a best-effort basis. Confirmed public repository issues will be prioritized based on severity, exposure, and reproducibility.

Private enterprise, customer-specific, or backend-service matters must be handled through the applicable commercial agreement and disclosure process.

## Safe Harbor

Good-faith, non-intrusive research focused on this public repository is welcome when it follows this policy. Activity that accesses private systems, disrupts services, extracts data, or targets third parties is not authorized.
