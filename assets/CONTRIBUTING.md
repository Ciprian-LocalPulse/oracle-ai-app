# Contributing

Thank you for your interest in Oracle AI App.

This repository is maintained as a public governance and presentation layer. Contributions must preserve the public/private boundary and keep the project suitable for enterprise, procurement, security, and legal review.

## Contribution Scope

Appropriate public contributions include:

- Typo fixes.
- Broken public link corrections.
- Documentation clarity improvements.
- Accessibility improvements for public static files.
- Non-sensitive governance improvements.

Do not submit:

- Secrets, credentials, API keys, tokens, or private configuration.
- Customer data, client references, or environment-specific details.
- Backend API logic, model prompts, payment logic, or token-generation flows.
- Operational deception logic, detection rules, telemetry schemas, or response procedures.
- Proprietary playbooks, business strategy, pricing, or acquisition material.
- Offensive security workflows, exploit instructions, or intrusive testing methods.

## Documentation Standards

Public documentation should be:

- Clear, factual, and procurement-safe.
- Defensive-security aware.
- CISO-friendly and legally conservative.
- Careful to separate public presentation materials from private enterprise delivery.
- Free from exaggerated security, financial, operational, or compliance claims.

Avoid aggressive, militarized, or legally risky language. Use precise wording that can be reviewed by enterprise security, procurement, and legal stakeholders.

## Pull Request Checklist

Before submitting a pull request, confirm that:

- The change is limited to public-safe documentation or static presentation.
- No private implementation detail is disclosed.
- No secrets or credentials are included.
- No customer or client-specific information is included.
- Public and private editions remain clearly separated.

## Maintainer Review

All contributions may be edited, rejected, or deferred at maintainer discretion. Enterprise requests, commercial discussions, private backend work, and transfer materials must be handled separately.
